1) Vue generale

Scope restreint: 2 multi agents seulement
User -> BrowserRouter -> NavBar -> [Dashboard/Analyse] ou [Axis 2 Chat] -> Backend specialise -> Reponse structuree -> Rendu UI

Routes conservees

  • / -> Dashboard
  • /agents -> Analyse multi-agents
  • /axis2-chat -> Chat Axis 2

Variables reseau utiles

  • VITE_API_BASE
  • VITE_WS_URL
  • VITE_AXIS2_API_BASE_URL

Principe

  • Isolation metier par parcours
  • Orchestration backend par domaine
  • Rendu front simple et actionnable

2) Multi Agent: Dashboard / Analyse

Composants: App.jsx + AgentAnalysis.jsx

Responsabilites

  • Dashboard KPI en temps reel
  • Filtres business et tendances
  • Execution multi-agents d'analyse
  • Recommandations business/marketing

APIs

  • GET /api/kpis
  • WS /ws/kpis
  • GET /api/smart-recommendations/last
  • POST /api/smart-recommendations
  • GET /api/agent-analysis/last
  • POST /api/agent-analysis

Agents impliques

  • KPI Analyst
  • Customer Profiler
  • Market Researcher
  • Business and Marketing Agent

3) Multi Agent: Axis 2 Chat

Composants: Axis2ChatTab.jsx + axis2Api.js

Responsabilites

  • Chat decisionnel multilingue
  • Gestion des sessions et historique
  • Activation RAG conditionnelle
  • Affichage des sources RAG

APIs

  • POST /chat
  • GET /sessions/{user_id}
  • GET /sessions/{user_id}/{session_id}/messages

Coordination interne

  • Router -> Decision route
  • RAG (si requis) -> grounding
  • Generation finale -> reponse + actions
  • Persistance session/profil
